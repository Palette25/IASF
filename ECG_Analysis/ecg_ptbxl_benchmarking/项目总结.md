## 一、项目背景与研究问题

本项目是论文 **“Deep Learning for ECG Analysis: Benchmarks and Insights from PTB-XL”** 的配套代码，实现了在大规模心电图数据集 PTB‑XL（以及 ICBEB 2018 挑战数据）上的系统化深度学习基准实验与评估。  
心电图（ECG）是临床中最常用的无创检查手段之一，但传统自动诊断算法往往依赖手工特征、覆盖的心律失常类型有限，且在大规模真实临床数据上的表现缺乏系统评估。本项目要解决的核心问题是：

- **在大规模临床 ECG 数据集上，深度学习模型到底能取得什么样的诊断性能？**  
- **不同网络结构（ResNet、Inception、RNN、Wavelet+NN 等）在不同任务（全部诊断语句、诊断类/子类/大类、形态、节律）上的表现如何？**  
- **如何提供一套统一的、可复现的实验框架，方便社区在 PTB‑XL / ICBEB 上进行公平对比和扩展？**

## 二、数据集与任务设置

### 1. PTB‑XL 数据集

- **来源**: PhysioNet 上公开的 PTB‑XL 数据集，包含 **21,799 条 12 导联 ECG**，来自 **18,869 名患者**。  
- **采样率**: 同一心电记录提供 500 Hz 与 100 Hz 两种采样率。  
- **标注**: 采用 SCP‑ECG 标准给出多条诊断语句，相关映射信息保存在 `scp_statements.csv` 中。

基于 PTB‑XL 的标注，本项目定义了多个子任务（对应不同的标签聚合方式）：

- **全部诊断语句（all statements, exp0）**：对所有 SCP 语句进行多标签预测。  
- **诊断语句（diagnostic, exp1）**：仅考虑标记为诊断的语句。  
- **诊断子类（diagnostic subclasses, exp1.1）**：将诊断语句聚合为更细粒度的子类。  
- **诊断大类（diagnostic superclasses, exp1.1.1）**：进一步聚合为更粗粒度的大类。  
- **形态（form, exp2）**：针对形态相关的语句进行分类。  
- **节律（rhythm, exp3）**：针对心律相关的语句进行分类。

### 2. ICBEB 2018 竞赛数据集

- **来源**: 2018 年 ICBEB 心律失常检测挑战赛的公开训练集。  
- **处理流程**: 原始 `.mat` 信号与 `REFERENCE.csv` 标签经过脚本 `code/utils/convert_ICBEB.py` 转换为：
  - 500 Hz 与 100 Hz 的 WFDB 格式记录（`records500/`、`records100/`）  
  - 对应的标签表 `icbeb_database.csv`，将挑战赛类别映射为统一的诊断标签。

在 ICBEB 上，项目主要进行“**所有诊断语句（all statements）**”场景下的多标签评估（实验名 `exp_ICBEB`）。

## 三、模型与方法

项目实现并对比了多种深度学习结构及一个基于小波的经典机器学习基线，主要包括：

### 1. Fastai 深度学习模型（`fastai_model`）

这些模型统一由 `code/models/fastai_model.py` 封装，基于 fastai v1 + PyTorch 实现：

- **ResNet 系列**：`resnet1d18/34/50/101/152` 与文献中 1D ResNet 变体 `resnet1d_wang`。  
- **Wide ResNet**：`wrn1d_22`。  
- **XResNet1d 系列**：`xresnet1d18/34/50/101/152` 及其 deep / deeper 变体，对一维信号做改进的 ResNet 家族。  
- **Inception1d**：一维版 Inception 网络，用多个不同长度的一维卷积核并行提取特征，适合 ECG 这种多时间尺度模式。  
- **FCN / Schirrmeister / Sen / BasicConv1d 等 1D CNN 结构**：涵盖不同深度与拓扑的卷积网络。  
- **RNN 系列**：`fastai_lstm`、`fastai_lstm_bidir`、`fastai_gru`、`fastai_gru_bidir`，对整段心电作为时间序列进行建模。

这些模型通过统一的 `ClassificationModel` 接口进行训练和推理，支持：

- `fit(X_train, y_train, X_val, y_val)`：训练  
- `predict(X)`：对任意样本集合输出多标签预测概率  
- 可配置的输入长度、学习率、批大小、是否 finetune 等参数。

### 2. Wavelet + 传统机器学习模型（`Wavelet+NN`）

在 `code/models/wavelet.py` 中实现：

- 利用小波分解、FFT、统计特征等构造人工特征；
- 使用如随机森林、MLP、Logistic 回归等传统分类器；
- 并引入简单的神经网络结构作对比。

这是论文中的一个重要 **非深度学习基线**，用于展示深度模型相对传统方法的优势。

### 3. 数据处理与标签聚合

核心逻辑在 `code/utils/utils.py`：

- **数据读取与缓存**：  
  - `load_dataset` 根据数据路径自动区分 PTB‑XL / ICBEB，并加载相应的 `*_database.csv`；  
  - 利用 WFDB 读取原始记录，并将转换后的 numpy 数组缓存为 `raw100.npy` / `raw500.npy`，避免反复从磁盘解码。

- **标签聚合（`compute_label_aggregations`）**：  
 结合 `scp_statements.csv` 中的诊断标记，将原始 SCP 语句聚合为不同的任务标签（诊断、子类、大类、形态、节律、all_scp 等）。

- **样本筛选与 one‑hot 编码（`select_data`）**：  
 通过 `MultiLabelBinarizer` 将多标签列表转成 one‑hot 矩阵，并按 `min_samples` 过滤掉过于稀有的类别。

- **信号标准化（`preprocess_signals`）**：  
 基于训练集的全局均值和方差对所有样本做标准化，并将标准化器保存下来以保证一致性。

### 4. 实验框架（`SCP_Experiment`）

`code/experiments/scp_experiment.py` 定义了统一的实验类 `SCP_Experiment`，负责：

- 加载与划分数据（按 `strat_fold` 做 10 折划分：1‑8 训练、9 验证、10 测试）；  
- 调用多个模型进行训练与预测；
- 生成简单基线（`naive`：用训练集标签均值作为概率预测）；
- 构建模型集合的简单 **ensemble**（对各模型预测求均值）；
- 对训练/验证/测试集分别进行评估并保存结果。

这样，所有模型与实验在统一框架下执行，方便横向比较和扩展。

## 四、评估指标与结果分析

### 1. 评估指标

主要指标由 `utils.evaluate_experiment` 实现：

- **macro AUC（macro_auc）**：宏平均 ROC AUC，是所有任务的核心指标；  
- **Fmax / F_beta_macro / G_beta_macro**：  
  - 对 ICBEB 挑战赛任务，项目实现了和官方评测一致的 `F_beta` 与 `G_beta` 分数；  
  - 通过在训练集上寻找最优阈值 `thresholds`，再在验证或测试集上计算对应指标。

评估模块还支持 **bootstrapping** 来估计指标置信区间，不过在默认设置与本次 Windows 适配中，多数情况下采用直接整体评估（`bootstrap_eval=False`），更高效也更稳定。

### 2. 实验效果概览

项目在 README 中给出了完整的排行榜，这里摘取几个关键结论（均为论文中的代表结果）：

- 在 PTB‑XL 各类任务上，**深度 CNN 模型（如 xresnet1d101、inception1d、resnet1d_wang）整体表现最好**：  
  - 所有语句（all statements）：AUC ≈ 0.92–0.93；  
  - 诊断语句 / 子类 / 大类：AUC 通常在 0.93 左右甚至更高；  
  - 节律任务上 AUC 可达 ~0.95 以上。
- **双向 LSTM 等 RNN 模型表现接近 CNN，但略逊一筹**，CNN 的卷积结构在 ECG 上更有效。  
- 传统的 **Wavelet+NN** 基线在所有任务上显著落后于深度 CNN / RNN，验证了端到端深度学习在大规模 ECG 上的优势。  
- 在 ICBEB 数据集上，xresnet1d101 等模型也取得非常有竞争力的结果（macro AUC 约 0.97），并给出了 F_beta 和 G_beta 的高分。

总体而言，项目通过统一框架，在多个任务、多种结构上系统对比，给出了 **深度学习做 ECG 自动诊断的高水准基线**。

## 五、项目结构与执行流程

### 1. 主要目录结构

- `code/`：核心代码  
  - `models/`：各类模型实现（fastai CNN/RNN、小波模型等）  
  - `utils/`：数据加载、预处理、评估、结果表生成等  
  - `experiments/`：`SCP_Experiment` 实验框架  
  - `configs/`：不同模型的配置（如 `fastai_configs.py`）  
  - `reproduce_results.py`：一键复现实验主脚本  
  - `evaluate_only.py`（本次适配新增）：只基于已有预测补充评估与汇总。
- `data/`：数据目录  
  - `ptbxl/`：PTB‑XL 原始数据与索引文件  
  - `ICBEB/`：转换后的 ICBEB 数据与索引文件  
- `output/`：实验输出  
  - `exp*/models/*/`：不同实验与模型的预测与结果  
  - `results_ptbxl.csv`、`results_icbeb.csv`：整体汇总表。

### 2. 完整执行流程概述

1. **安装依赖与环境准备**  
   - 原始项目提供 `ecg_env.yml` 供 conda 创建环境；  
   - 本次在 Windows 上额外给出 `requirements.txt` 与若干兼容性修正（PyTorch/fastai/TensorFlow/protobuf 等版本组合）。

2. **数据准备**  
   - `get_datasets.sh` / `get_datasets_win.py` 下载并整理 PTB‑XL 与 ICBEB；  
   - `convert_ICBEB.py` 将 ICBEB 原始 `.mat` 转为 WFDB 格式，并生成 `icbeb_database.csv`。

3. **复现实验**  
   - `python code/reproduce_results.py`：  
     - 依次运行 PTB‑XL 上的 `exp0–exp3` 以及 ICBEB 上的 `exp_ICBEB`；  
     - 对每个实验和模型进行训练、预测与评估；  
     - 生成 summary 表（Markdown 与 CSV）。

4. **只补评估、不重新训练**（本次适配新增）  
   - `python code/evaluate_only.py`：  
     - 假设预测文件 `y_*_pred.npy` 已存在，只重新跑 `evaluate()`；  
     - 生成/更新各模型的 `tr_results.csv` / `val_results.csv` / `te_results.csv` 与整体结果表。

5. **自定义模型基准**  
   - 在 `code/models/your_model.py` 中按指定接口实现自己的模型；  
   - 在 `code/configs/your_configs.py` 中加入对应 config；  
   - 在 `scp_experiment.py` 的 `perform()` 中添加新模型类型分支；  
   - 使用修改后的 `reproduce_results.py` 进行统一评估和对比。

## 六、项目贡献与可扩展性

本项目的主要贡献可以概括为：

- **首批在大规模临床 ECG 数据集 PTB‑XL 上系统对比多种深度学习架构的公开实现**，并提供完整可复现实验。  
- 提供了一套 **通用的 ECG 深度学习基准框架**：从数据预处理、标签聚合、模型训练到评估与结果汇总，结构清晰、易于扩展。  
- 同时覆盖 **PTB‑XL 与 ICBEB 两个数据集**，并在 ICBEB 上复现了挑战赛官方指标，有利于迁移与对比研究。  
- 对研究者和工程师而言，该仓库既可作为 **论文复现基线**，也可以作为 **开发新 ECG 模型与评估方法的实验平台**。

## 七、小结

总体来看，本项目通过严谨的实验设计、丰富的模型对比和清晰的工程实现，系统回答了“**深度学习在 ECG 自动分析上的能力边界在哪里**”这一关键问题，同时为后续工作提供了：

- 高质量公开代码与结果基线；  
- 易于扩展的实验框架；  
- 在临床 ECG 场景下可直接借鉴的建模与评估实践。

- 针对特定疾病或标签子集的精细化研究；  
- 将自己医院/实验室的 ECG 数据映射到 PTB‑XL/ICBEB 的标签体系进行迁移学习。
